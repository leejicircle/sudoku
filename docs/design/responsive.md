# 🖌 반응형 브레이크포인트 가이드 (Responsive Design)

> 모든 화면의 반응형 동작을 통합 정의한다.
> 각 페이지 문서의 반응형 섹션은 이 문서를 참조한다.

---

## 1. 브레이크포인트 정의

Tailwind CSS 기본 브레이크포인트를 따른다.

| 이름 | 약어 | 최소 너비 | 대상 디바이스 |
|------|------|----------|-------------|
| **xs** | — | 0px | 소형 모바일 (SE, Galaxy Fold 등) |
| **sm** | `sm:` | 640px | 대형 모바일 / 소형 태블릿 |
| **md** | `md:` | 768px | 태블릿 (iPad Mini 등) |
| **lg** | `lg:` | 1024px | 데스크톱 / 태블릿 가로 |
| **xl** | `xl:` | 1280px | 대형 데스크톱 |

### 1.1 주요 기준점

| 너비 | 의미 |
|------|------|
| 320px | 최소 지원 너비 (iPhone SE) |
| 375px | 최적 모바일 (iPhone 12/13/14/15 기본) |
| 390px | iPhone 14 Pro / 15 Pro |
| 430px | iPhone 14/15 Pro Max |
| 768px | 태블릿 세로 |
| 1024px | 태블릿 가로 / 데스크톱 |

---

## 2. 모바일 우선 전략

모든 스타일은 **모바일 기준(375px)**으로 작성하고, 큰 화면으로 확장한다.

```css
/* 기본: 모바일 (375px) */
.component { ... }

/* 태블릿 이상 */
@media (min-width: 768px) { ... }

/* 데스크톱 이상 */
@media (min-width: 1024px) { ... }
```

Tailwind 클래스 패턴:
```html
<div class="p-4 md:p-6 lg:p-8">
  <!-- 모바일: 16px → 태블릿: 24px → 데스크톱: 32px -->
</div>
```

## 3. 게임 보드 반응형

### 3.1 셀 크기 스케일링

| 뷰포트 | 셀 크기 | 보드 너비 | CSS 변수 |
|--------|--------|----------|---------|
| < 360px | 36px | 340px | 커스텀 |
| 360px ~ 374px | 38px | 358px | 커스텀 |
| 375px ~ 639px | 40px | 376px | `--cell-size` |
| 640px ~ 767px | 44px | 412px | 보간 |
| 768px ~ 1023px | 48px | 448px | `--cell-size-md` |
| ≥ 1024px | 56px | 520px | `--cell-size-lg` |

### 3.2 보드 컨테이너 전략

```css
.board-container {
  /* 모바일: 보드가 화면 너비에 꽉 참 */
  width: 100%;
  max-width: calc(var(--cell-size) * 9 + var(--board-gap-thin) * 6 + var(--board-gap-thick) * 2 + var(--board-border-width) * 2);
  margin: 0 auto;
}

/* 태블릿+: 고정 크기 + 중앙 정렬 */
@media (min-width: 768px) {
  .board-container {
    --cell-size: 48px;
  }
}

@media (min-width: 1024px) {
  .board-container {
    --cell-size: 56px;
  }
}
```

### 3.3 초소형 뷰포트 대응 (< 360px)

320px까지 지원하기 위한 폴백:

```css
@media (max-width: 359px) {
  :root {
    --cell-size: 34px;        /* 최소 34px (터치 한계) */
    --text-cell: 20px;        /* 셀 숫자 축소 */
    --text-cell-memo: 8px;    /* 메모 숫자 축소 */
    --numpad-button-size: 42px;
    --board-padding: 8px;
  }
}
```

## 4. 레이아웃 반응형

### 4.1 Header

| 뷰포트 | 높이 | 변경사항 |
|--------|------|---------|
| < 640px | 56px | 기본 모바일 |
| ≥ 768px | 64px | 패딩 좌우 24px, 콘텐츠 max-w: 768px |
| ≥ 1024px | 64px | max-w: 1024px, BottomNav 링크 통합 |

### 4.2 BottomNav

| 뷰포트 | 동작 |
|--------|------|
| < 375px | 라벨 숨김 (아이콘만) |
| 375px ~ 767px | 기본 (아이콘 + 라벨) |
| ≥ 768px | **숨김** → Header에 네비게이션 통합 |

### 4.3 콘텐츠 영역

| 뷰포트 | max-width | 패딩 |
|--------|----------|------|
| < 640px | 100% | 16px |
| 640px ~ 767px | 100% | 24px |
| ≥ 768px | 768px (중앙) | 32px |
| ≥ 1024px | 1024px (중앙) | 40px |

## 5. 페이지별 반응형 요약

### 5.1 홈 (스테이지 선택)

| 뷰포트 | 카드 레이아웃 | 카드 높이 |
|--------|-------------|----------|
| < 640px | 1열 스택 | 80px |
| ≥ 768px | 2×2 그리드 (gap: 16px) | 100px |
| ≥ 1024px | 2×2 그리드 (max-w: 600px 중앙) | 100px |

### 5.2 게임 플레이

| 뷰포트 | 보드 | 도구바 | 숫자패드 |
|--------|------|--------|---------|
| < 360px | 셀 36px, 도구 아이콘만 | 4열 | 5열 (44px 버튼) |
| 375px ~ 767px | 셀 40px | 4열 (아이콘+라벨) | 5열 (48px 버튼) |
| ≥ 768px | 셀 48px | 가로 확장 | 가로 확장 |
| ≥ 1024px | 셀 56px | **2컬럼**: 보드 좌측, 도구+패드 우측 | 우측 패널 |

#### 데스크톱 2컬럼 레이아웃 (≥ 1024px)

```
┌────────────────────────────────────────────┐
│  Header                                    │
├──────────────────┬─────────────────────────┤
│                  │                         │
│                  │    도구바               │
│    보드 (56px)    │    ↩ 되돌리기 ✕ 지우기  │
│    520px         │    ✏ 메모    💡 힌트   │
│                  │                         │
│                  │    숫자패드              │
│                  │    [1][2][3][4][5]      │
│                  │    [6][7][8][9][⌫]     │
│                  │                         │
└──────────────────┴─────────────────────────┘
```

### 5.3 랭킹

| 뷰포트 | 포디움 | 리스트 |
|--------|--------|--------|
| < 375px | 높이 150px, 아바타 축소 | 기본 |
| 375px ~ 767px | 높이 180px | 기본 |
| ≥ 768px | 리스트 max-w: 600px 중앙 | 기본 |
| ≥ 1024px | 포디움 + 리스트 나란히 (2컬럼) | max-w: 800px |

### 5.4 로그인

| 뷰포트 | 변경사항 |
|--------|---------|
| < 375px | 버튼 max-w: 280px, 로고 56px |
| 375px ~ 767px | 기본 (중앙 정렬) |
| ≥ 768px | 카드 UI (400px, 배경+그림자) |
| ≥ 1024px | 좌측 일러스트 + 우측 로그인 (2컬럼) |

### 5.5 클리어 모달

| 뷰포트 | 모달 너비 | 패딩 |
|--------|----------|------|
| < 375px | calc(100vw - 32px) | 24px 16px |
| 375px ~ 767px | min(calc(100vw - 48px), 360px) | 32px 24px |
| ≥ 768px | 420px | 40px 32px |

## 6. 터치 vs 마우스 인터랙션

### 6.1 감지

```css
/* 터치 디바이스 */
@media (hover: none) and (pointer: coarse) {
  /* 호버 효과 제거, 터치 피드백 강화 */
}

/* 마우스/트랙패드 디바이스 */
@media (hover: hover) and (pointer: fine) {
  /* 호버 효과 활성화 */
}
```

### 6.2 디바이스별 차이

| 인터랙션 | 터치 (모바일) | 마우스 (데스크톱) |
|----------|-------------|-----------------|
| 셀 선택 | 탭 | 클릭 |
| 셀 호버 | 없음 | 약한 하이라이트 (`--accent` / 50%) |
| 숫자 입력 | 숫자패드 탭 | 키보드 1~9 또는 패드 클릭 |
| 셀 이동 | 탭 | 화살표 키 또는 클릭 |
| 버튼 호버 | 없음 (active만) | 배경색 변경 |
| 스크롤 | 스와이프 | 마우스 휠 |
| 숫자패드 피드백 | `navigator.vibrate(10)` | 없음 |

## 7. 다크 모드 반응형

다크 모드는 브레이크포인트와 독립적으로 동작한다.

### 7.1 전환 방식

| 속성 | 값 |
|------|-----|
| 기본 | 시스템 설정 따름 (`prefers-color-scheme`) |
| 수동 | 향후 설정에서 토글 (v2) |
| 클래스 | `.dark` (html 요소) |

### 7.2 다크 모드 추가 고려사항

| 요소 | Light → Dark 변경 |
|------|-------------------|
| 그림자 | 약한 그림자 → 보더/글로우 |
| 보드 외곽 | 진한 보더 → 밝은 보더 |
| 이미지 | — | 필요 시 `brightness(0.8)` |
| 아이콘 | 색상 자동 전환 (CSS 변수 기반) |

## 8. 성능 고려사항

### 8.1 이미지

| 전략 | 설명 |
|------|------|
| 아이콘 | Lucide React (SVG, 트리 쉐이킹) — 이미지 없음 |
| 아바타 | `next/image` + 작은 사이즈 (36~56px) |
| 로고 | SVG 또는 텍스트 — 별도 이미지 없음 |

### 8.2 폰트

| 전략 | 설명 |
|------|------|
| Geist Sans/Mono | `next/font/google`로 로컬 로드 (이미 설정됨) |
| font-display | swap (FOUT 허용, 빠른 렌더링) |

### 8.3 애니메이션

| 전략 | 설명 |
|------|------|
| `prefers-reduced-motion` | 모션 감소 설정 시 애니메이션 제거/축소 |

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 9. 뷰포트 단위 참조

| 단위 | 사용처 | 설명 |
|------|--------|------|
| `dvh` | 전체 레이아웃 높이 | 동적 뷰포트 (주소바 포함/미포함) |
| `svh` | 모달 최대 높이 | 최소 뷰포트 (안전 영역) |
| `vw` | 보드 fallback 크기 | 뷰포트 너비 비율 |
| `env(safe-area-inset-*)` | Header/BottomNav 패딩 | 노치/홈바 영역 |
